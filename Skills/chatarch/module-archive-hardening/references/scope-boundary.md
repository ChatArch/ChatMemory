# Scope Boundary

Promote a skill to `Skills/chatarch/` only when the reusable value is a stable ChatArch workflow.

Keep the content out of ChatArch skills when:

- it depends on a private site, a personal account, or local secrets;
- it is a one-off operational workaround;
- it is not yet stable enough to reuse across repositories.

Promote the generalized workflow instead:

- archive-to-package review;
- CLI surface cleanup;
- package release governance;
- provider discovery and test rules.
